# REVIEW: docs / skills alignment

Comprehensive audit of plugins/mlld/skills/ and plugins/mlld/examples/ against docs/src/atoms/ (source of truth). Needs adam's review before any changes are made.

## Cross-cutting issues (all skills)

1. **`@claudePoll` used as primary/only LLM pattern** ‚Äî Every skill file uses `@claudePoll` exclusively. The current convention is `@claude(prompt, config)` from `import { @claude } from @mlld/claude` as the primary pattern, with `@claudePoll` as a secondary variant for file-output polling.

2. **`env` blocks instead of `box`** ‚Äî The agents skill explicitly uses `env` blocks (agents/patterns.md line 84+). All skills should reference `box` as the primitive for scoped execution contexts.

3. **No mention of `box` anywhere** ‚Äî Zero usage of the `box` directive across all skills, despite it being mlld's primitive for isolation, credentials, and tool scoping.

4. **Missing `exe llm` label** ‚Äî None of the exe definitions in skills carry the `llm` label, which is required for the runtime to process `config.tools` and populate `@mx.llm`.

5. **Missing `import` statements** ‚Äî Most examples use `@claudePoll` without showing the required `import { @claudePoll } from @mlld/claude`.
