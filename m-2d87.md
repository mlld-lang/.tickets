# [User Request] Document sealed credentials security rationale

The auth/sealed credentials docs explain the mechanics but not the security rationale prominently enough. A security professional evaluated mlld and concluded that sealed credentials 'not accessible inside js blocks' was a bug that 'undermines the security model' -- the exact opposite of reality. The docs need to lead with WHY credentials are isolated from the interpolation namespace: prompt injection attacks that manipulate template interpolation cannot exfiltrate credentials because they don't exist in that scope. This should be front and center in docs/src/atoms/config/14-auth.md and docs/src/atoms/config/09-policy--auth.md, not buried. If a security person misreads this, our docs failed.
