# [User Request] @validate(schema) transformer for structured LLM output validation

Add a @validate builtin transformer that checks parsed JSON against a schema and produces structured errors suitable for guard retry hints. Covers the same ground as Pydantic ResponseModel + instructor: type checking (string, number, boolean), enum constraints (array of allowed values), nested objects, arrays of typed objects, required vs optional fields ("string?"), min/max on numbers. Should compose with existing guard retry system: @output | @parse.llm | @validate(@schema). On failure, produces LLM-friendly error messages for retry hints. Consider using ajv under the hood for JSON Schema, or a lighter custom implementation for a simplified schema shorthand. Schema shorthand concept: { summary: "string", sentiment: ["positive", "negative", "neutral"], entities: [{ name: "string", confidence: "number" }] }. Origin: Nexus team evaluation identified this as the primary gap preventing adoption.
