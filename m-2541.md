# Function MCP bridge not exposing exe refs as tools to Claude CLI

QA tests 6a and 7a fail: Claude CLI says 'I don't have a tool called double' when `tools: [@double]` is passed in a box. The function MCP bridge (FunctionMcpBridge / FunctionRouter) should expose mlld exe refs as MCP tools, but Claude CLI doesn't see them.

Test 6a: exe ref only — `tools: [@double]`
Test 7a: mixed string + exe ref — `tools: ["Read", @uppercase]`

Both fail with Claude saying the tool isn't available. The VFS string tools (Read, Write, Glob, Grep) work correctly (tests 4a, 4b, 5a pass). The issue is specific to function tool exposure.

Investigation areas:
- FunctionMcpBridge (interpreter/env/executors/function-mcp-bridge.ts) — is it starting the server?
- FunctionRouter (cli/mcp/FunctionRouter.ts) — is it registering tools correctly?
- call-mcp-config.ts — is the combined MCP config pointing to both VFS and function servers?
- Are the function tool names in `--allowedTools` matching what the function bridge exposes?

QA test file: tmp/qa-claude-v3.mld.md (tests 6a, 7a)
Module under test: modules/claude/index.mld


## Notes

**2026-03-05T20:26:06Z** ## Root Cause Found

The issue is in `interpreter/utils/node-interop.ts:toJsValue()`.

### Chain of events

1. `@toolbridge` is a `nodeFunction` (TypeScript builtin)
2. When calling a nodeFunction, arguments go through `toJsValue()`
3. `toJsValue` recursively processes arrays (line 387): `value.map(item => toJsValue(item))`
4. Each array item that is an ExecutableVariable hits line 390-391: throws "Executable values are not valid node arguments"

### Why it's silent in QA

The `@claude` module wraps `@toolbridge` in a `when` expression. The error is caught by when, which falls through to `*` (no tools). Claude runs without tool access.

### Fix

`toJsValue` should pass through ExecutableVariables instead of throwing. `@toolbridge` is designed to receive them — `resolveToolInput` in `call-mcp-config.ts:231` checks `isExecutableVariable()`.

### Key file

`interpreter/utils/node-interop.ts:390-391` — the throw that blocks exe refs from reaching `@toolbridge`.
