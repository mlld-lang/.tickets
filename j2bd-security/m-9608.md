# Fix mcp-policy.md: remove unimplemented policy.sources claim

## Problem

mcp-policy.md lines 48-59 claim `sources: { 'src:mcp': 'untrusted' }` works for auto-classifying MCP data as untrusted. This field is NOT implemented in PolicyConfig (confirmed in core/policy/union.ts lines 64-77).

## Fix Required

Replace the "Combining with source classification" section with honest documentation:

1. Remove the non-working `sources:` example
2. Explain the manual workaround: `var untrusted @mcpData = @mcp.tool.call()`
3. Note that automatic source classification is a planned feature

## Example Fix

Replace lines 48-59 with:

```
**Manual trust labeling:**

Currently, MCP data gets `src:mcp` taint but not automatic trust classification. Use manual labeling:

```mlld
var untrusted @mcpData = @mcp.github.listIssues()
```

Now `@mcpData` has both `src:mcp` taint AND the `untrusted` label, so `no-untrusted-destructive` applies.
