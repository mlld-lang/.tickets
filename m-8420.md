# Add parameter shadowing warning to mlld validate

Warn when function parameter names could shadow variables in calling scopes. Example: Parameter 'result' in @logItemDone() shadowed caller's @result variable, causing subtle corruption bug (m-7cd0).

## Acceptance Criteria

- mlld validate warns when exe parameter name matches common variable patterns
- Warning suggests more specific names (e.g., 'status' instead of 'result')
- Can be suppressed with config if intentional

## Notes

**2026-01-30**: Assessed as not implemented. Tested `mlld validate` with exe function using `@result` parameter - no warning produced. Note: mlld validate DOES warn about `let` variable shadowing in nested scopes (different issue), but not about function parameter naming.
