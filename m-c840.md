# Auto-create VFS for any box block that uses file/files directives

## Problem

`box @cfg [...]` where `@cfg` is a config-only object (e.g. `{ tools: ["Bash", "Read"] }`) does not create a VFS or ShellSession. If a `file` directive appears inside the block, the write goes to an orphaned workspace that shell commands can't see.

```mlld
>> BROKEN: file writes to VFS but run cmd has no ShellSession
var @cfg = { tools: ["Bash", "Read"] }
box @cfg [
  file "x.txt" = "hello"
  run cmd { cat x.txt }   >> file not found
]

>> WORKS: anonymous box auto-creates VFS
box [
  file "x.txt" = "hello"
  run cmd { cat x.txt }   >> hello
]
```

## Fix

Any box block should auto-create an anonymous VFS + ShellSession when no workspace is already active. The three forms should converge:

- `box [...]` — already creates VFS (works today)
- `box @resolver [...]` — uses resolver's VFS (works today)
- `box @cfg [...]` / `box { tools: [...] } [...]` — should also create VFS

In `evaluateBox()`: after resolving config, if no workspace was provided (no `fs:` field, no resolver), create `VirtualFS.empty()` and push it as the active workspace. Apply config restrictions (tools, mcps, net) on top.

## Interpreter change

In `interpreter/eval/box.ts`, in the config resolution path where `workspace` is null, create one:

```
if (!workspace) {
  workspace = { type: 'workspace', fs: VirtualFS.empty(), descriptions: new Map() };
}
scopedEnv.pushActiveWorkspace(workspace);
```

## Doc changes

`docs/src/atoms/config/13-box--blocks.md` — the anonymous workspace block section currently says:

> `box [ ... ]` creates an anonymous in-memory workspace for the block.

Change to:

> All box blocks have an in-memory workspace. `box [...]`, `box @config [...]`, and `box @resolver [...]` all support `file`/`files` directives and shell commands against VFS.

`docs/src/atoms/config/11-box--directive.md` — add a note that config-form boxes also get VFS:

> `box { tools: ["Read", "Bash"] } [...]` restricts tools and provides an in-memory workspace. Use `box { fs: @resolver } [...]` to bind an existing resolver's filesystem instead.

## Test

```mlld
var @cfg = { tools: ["Bash", "Read"] }
box @cfg [
  file "x.txt" = "config-box-vfs"
  let @r = run cmd { cat x.txt }
  => @r
]
>> expected: config-box-vfs
```
