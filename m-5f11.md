# mlld validate should catch undefined variables passed to function calls

When an exe wrapper passes an optional/undefined parameter to an inner function, `mlld validate` reports no errors but the runtime throws:

```
Error: Undefined variable '@timeout' passed to function @resumeSession
```

**Reproduction**: An outer exe declares `timeout` as a parameter but callers don't pass it. The outer exe passes `@timeout` to an inner function. Validate passes, runtime fails.

**Example**:
```mlld
exe @inner(x, timeout) = sh { echo $timeout }
exe @outer(x, timeout) = [
  let @result = @inner(@x, @timeout)
  => @result
]
>> Caller omits timeout â€” validate should warn, runtime throws
var @r = @outer("hello")
```

**Expected**: `mlld validate` should warn about potentially undefined variables being passed as arguments to function calls, especially when the caller can omit optional trailing parameters.

## Acceptance Criteria

1. `mlld validate` produces a warning when an optional parameter is passed through to another function without a default
2. No false positives for parameters that are always provided by callers
