# Fix misleading doc: pipelines-context.md says @mx.stage = stage name

## What's wrong

In `docs/src/atoms/pipelines/pipelines-context.md`, the docs claim `@mx.stage` returns the current stage name. It actually returns an integer index (1, 2, 3...).

## Fix

Find the `@mx.stage` description in pipelines-context.md and change it to say it returns the 1-based integer index of the current pipeline stage, not a name string. Add an example showing the actual output.
