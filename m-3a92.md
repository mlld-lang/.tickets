# [User Request] Document guard system capabilities for LLM validation patterns

The guard system's capabilities for LLM output validation are not discoverable enough. An external evaluation listed rate limiting, retry logic, and custom validators as features mlld is MISSING -- all of which guards already provide. Need documentation and/or cookbook examples showing: (1) after-guard validation on LLM output with retry and hint feedback, (2) rate limiting via for parallel(cap, rateMs), (3) composable guard modules for reusable validation, (4) content inspection patterns. These should be findable by someone evaluating mlld for LLM orchestration.
