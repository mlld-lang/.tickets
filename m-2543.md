# Maybe: Make Imported var tools Portable for Metadata Consumers, Not Execution

Imported var tools should carry portable metadata for read-only consumers such as auth validation, planning, schemas, and introspection, without changing execution semantics. The current auth fix only preserves AuthorizationToolContext under toolCollectionMetadata.auth. This follow-up would decide which additional metadata consumers should read portable collection metadata directly before falling back to env lookup.

## Acceptance Criteria

- Define the non-execution consumer boundary for imported `var tools` metadata.
- Use an extensible `toolCollectionMetadata` envelope and document which fields are portable.
- Keep actual tool execution env-bound; do not make imported tool collections executable without live runtime bindings.
- Add focused tests for each adopted metadata consumer.
