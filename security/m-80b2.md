# Module imports should be idempotent â€” double import causes guard re-registration error

Importing the same module twice (e.g., once for a named guard import and once for a policy import) throws 'Guard with name X already exists'. Module evaluation should be cached so that re-importing the same module reuses the previous evaluation result rather than re-executing side effects like guard registration.
